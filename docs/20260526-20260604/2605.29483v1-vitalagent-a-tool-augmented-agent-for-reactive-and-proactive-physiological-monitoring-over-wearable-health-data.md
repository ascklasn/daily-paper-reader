## 摘要
可穿戴设备能够持续监测诸如心电图（ECG）和光电容积脉搏波（PPG）等生理信号，但现有的移动健康（mHealth）系统大多局限于特定任务的预测流程或对静态摘要的反应式问答。它们缺乏支持时间推理、持续性生理上下文以及对长期信号流进行主动监测的能力。我们提出了VitalAgent，这是一种增强工具型智能体框架，用于基于ECG/PPG的移动健康，同时支持反应式问答和主动式监测。VitalAgent建立在纵向生理记忆和增强工具型推理接口之上，能够对原始信号进行动态计算。我们进一步介绍了VitalBench，这是一个纵向生理监测基准数据集，包含用于反应式问答的1,862个问答对，以及用于主动式监测的90.2小时连续ECG/PPG记录，涵盖了心脏、身体活动和压力相关任务。实验表明，VitalAgent在反应式评估中比基于提示和ReAct的基线方法提升了超过30%，并且支持对长期生理信号的主动警报监测，突显了动态工具使用和长期生理监测的重要性。

## Abstract
Wearable devices enable continuous monitoring of physiological signals such as ECG and PPG, but existing mHealth systems are largely limited to task-specific prediction pipelines or reactive question answering over static summaries. They lack the ability to support temporal reasoning, persistent physiological context, and proactive monitoring over long-term signal streams. We propose VitalAgent, a tool-augmented agentic framework for ECG/PPG-based mHealth that supports both reactive question answering and proactive monitoring. VitalAgent is built on a longitudinal physiological memory and a tool-augmented reasoning interface that enables dynamic computation over raw signals. We further introduce VitalBench, a longitudinal physiological monitoring benchmark dataset comprising 1,862 QA pairs for reactive question answering and 90.2 hours of continuous ECG/PPG recordings for proactive monitoring, covering cardiac, physical activity, and stress-related tasks. Experiments demonstrate that VitalAgent achieves over 30% improvement over prompt-based and ReAct baselines in reactive evaluation and supports proactive alert monitoring over long-term physiological signals, highlighting the importance of dynamic tool use and long-term physiological monitoring.

## 1. 核心问题与整体含义（研究动机与背景）
当前移动健康（mHealth）系统主要依赖任务特定的预测流水线或对静态摘要的反应式问答，无法支持对长期信号流的时间推理、持续生理上下文跟踪以及主动监测。例如，用户无法自然询问“我的静息心率最近是否上升？”或系统无法在用户未提问时主动发现异常。因此，论文旨在构建一个统一的智能体框架，同时支持：
- **反应式问答**（根据用户查询动态分析原始信号）
- **主动监测**（持续监控信号并触发告警）

## 2. 方法论：核心思想与关键技术细节
### 整体架构
VitalAgent 由三个核心组件构成：
- **生理记忆（Physiological Memory）**：存储历史信号窗口、推导出的生理状态（如心率、HRV、节律标签）以及过往告警记录。采用时间索引结构，用于跨时查询。
- **统一推理层（Unified Reasoning Layer）**：包含 LLM 规划器（基于用户查询或生理状态选择工具）、证据验证模块（检查计划完整性、工具成功性、字段存在性、跨工具一致性）和响应生成器。若验证失败，触发重规划。
- **结构化工具接口（Structured Tool Interface）**：包含 29 个工具，分为五类：信号分析工具（13个，执行确定性的心率/HRV/节律计算）、数据处理工具（3个，处理数据集差异）、记录查找工具（6个，获取元数据）、主动上下文工具（5个，检索历史告警）、医学知识工具（2个，接入MedlinePlus/PubMed）。

### 设计原则
- **动态工具选择**：不同于静态预计算特征，VitalAgent 根据每个查询或当前状态按需调用工具，实现查询依赖的生理推理。
- **信号计算与语言推理解耦**：所有低层特征（如心率）通过工具确定性计算，LLM 只负责规划、整合和响应生成。

### 主动监测流程
- 输入传感器数据被切分为 10 秒非重叠窗口。
- 规则检测器基于 RR 间期变异性特征（变异系数、熵、转向点比）触发三种告警：极端心动过缓、极端心动过速、持续心动过速（5分钟滑动窗口）。
- 每 20 个窗口还调用 LLM judge 判断是否应额外告警，以减少假阴性。

## 3. 实验设计
### 数据集与基准
- **VitalBench**：自建基准，基于四个公开数据集：
  - Icentia11k（ECG，节律标注）
  - AF-PPG-ECG（PPG+ECG，AF标注）
  - PPG-DaLiA（PPG，活动与心率）
  - WESAD（PPG，压力标签）
- 反应式部分：1862个QA对，分 Tier A（单窗口即时问题，906个）和 Tier B（多窗口聚合/比较问题，956个），类型包括 verify/choose/query。
- 主动监测部分：90.2小时连续ECG/PPG（69名患者，32,475个10秒窗口，47个异常AF片段）。

### 对比方法
- **Health-LLM**：基于预计算特征的提示方法（zero-shot, few-shot, few-shot CoT）
- **LifeAgent***：工具调用智能体（确定性计算工具）
- **PHIA**：ReAct风格健康智能体（代码执行作为工具）
- 设置两种上下文条件：**leakage-free**（信号推导特征）和 **oracle**（直接使用真实标注作为理想上界）

### 评估指标
- 反应式：准确率（Overall、Verify、Choose、Query 子指标）
- 主动式：每小时误报数（FAR/h）、告警延迟（从异常起始到首次告警的时间）
- 还需报告 AF 节律分类的灵敏度、特异性、准确率（用于验证信号分析工具质量）

- 充分性评估：覆盖多个数据集、多个任务类型、多个基线、消融、错误分析，实验设计较为系统。但注意仅有四个来源数据集，且非实时部署环境，存在局限性。

## 6. 主要结论与发现
1. **反应式问答提升显著**：VitalAgent 在 leakage-free 设置下，Tier A 准确率 86.0%，Tier B 55.7%，比最佳基线（LifeAgent*）分别高出 31.5% 和 25.7%。尤其在 Tier B Query 类型上，其他基线几乎失败（0.443 vs 0.0），体现动态工具对长时间聚合推理的优势。
2. **主动监测可行**：规则+LLM judge 可将告警延迟从 220 秒降至 105 秒，但 FAR/h 从 1.81 升至 2.95，体现延迟-准确率的权衡。AF 分类灵敏度 75.5%，特异性 57.9%。
3. **动态工具选择至关重要**：消融实验显示，移除规划器（随机选工具）导致 Tier A 准确率从 86.0% 降至 66.4%，Tier B 从 55.7% 降至 34.2%。
4. **验证与重规划贡献较小**（<1.2%），因需要重规划的样本极少（0.4%），但作为安全网仍有价值。
5. **失败模式**：75.4% 失败源于答案不匹配（错误聚合或推理），24.6% 是工具选择错误（特别在房颤/节律问题中难以区分时间范围）。压力检测最困难。

## 7. 优点
- **统一框架**：首次将反应式问答与主动监测整合在同一智能体架构中，共享记忆和工具，一致性强。
- **动态工具使用**：按需调用信号分析工具，避免静态摘要的信息损失，提高了查询依赖推理的准确性。
- **模块化与可解释性**：信号计算与语言推理解耦，所有低层特征确定性计算，易于调试和验证。
- **纵向记忆支持**：存储原始信号、推导状态和告警历史，支持跨时间比较和上下文感知。
- **评估全面**：构建了专门的纵向基准（VitalBench），覆盖多种模态、任务和时间粒度，并设置了有意义的对比基线。
- **开源实践**：代码公开，可复现。

## 8. 不足与局限
- **模态覆盖有限**：仅使用 ECG 和 PPG，未考虑呼吸、皮肤温度、EDA 等其他可穿戴信号。
- **主动监测规则固化**：使用预定义的临床阈值与简单特征，缺乏自适应或个性化告警策略，可能不够精准。
- **基准环境与现实差距**：VitalBench 源于公开数据集，信号质量较好；真实部署可能面临噪声、缺失数据、设备异质性等问题。
- **压力检测性能差**：WESAD 压力查询是主要失败来源，提示需要更先进的信号分析工具来捕捉细微生理变化。
- **时序聚合错误**：在“最高心率”等聚合查询中易出错，说明时间推理的数值计算仍具挑战。
- **伦理与安全**：作者明确声明该系统非诊断工具，仅作决策支持；但纵向记忆涉及隐私风险，且 LLM 可能产生错误或过度自信的答案。
- **评估规模有限**：仅四个数据集，且未进行跨人群或真实用户研究。
