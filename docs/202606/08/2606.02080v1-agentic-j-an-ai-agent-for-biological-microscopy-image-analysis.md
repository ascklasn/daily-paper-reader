## 摘要
生物图像分析越来越需要在异构工具、编程环境和领域知识之间进行整合，而很少有研究人员能同时掌握所有这些内容。我们推出了Agentic-J，一个容器化的多智能体AI助手，主要面向ImageJ/Fiji，使生物学家能够用自然语言指定分析任务，从细胞核分割、细胞追踪到多条件量化。该智能体生成可执行脚本，并组织成有文档记录的项目结构，因此每个分析决策都可追溯，工作流程可复现或共享。专门的子智能体负责插件管理、代码生成、调试、质量保证和统计报告。在本文中，我们介绍了该系统的设计，演示了真实的生物显微镜图像分析工作流程，并详细说明了技术实现。

## Abstract
Biological image analysis increasingly demands integration across heterogeneous tools, programming environments, and domain knowledge that few researchers can command simultaneously. We present Agentic-J, a containerised, multi-agent AI assistant, primarily for ImageJ/Fiji that enables biologists to specify analysis tasks in natural language, from nuclei segmentation and cell tracking to multi-condition quantification. The agent generates executable scripts organised into a documented project structure, so every analysis decision is traceable and the workflow can be reproduced or shared. The specialised sub-agents handle plugin management, code generation, debugging, quality assurance, and statistical reporting. In this paper we introduce the system's design, demonstrate real biological microscopy image analysis workflows, and detailed the technical implementation.

## 2. 论文提出的方法论

### 2.1 核心思想
- **多智能体架构**：系统由一组专业子智能体（supervisor、coder、debugger、plugin manager、plotting agent、QA agent）协同工作。用户通过浏览器中的聊天界面与 supervisor 交互，supervisor 根据科学目标将任务委派给相应子智能体。
- **容器化与沙箱**：整个系统封装在 Docker 容器中，包含预配置的 Fiji 分发版、精选插件集、Cellpose 模型权重、Conda 管理的 Python 环境（pyimagej、Qdrant、LangGraph）。智能体文件系统访问限制在专用 `data/` 目录，所有脚本生成和执行均在容器内进行。
- **知识库与检索增强生成（RAG）**：系统维护一个 Qdrant 向量集合，包含开源教材、社区最佳实践指南、插件文档等。同时拥有“技能”文件系统（SKILL.md）和“食谱/错误”数据库，使智能体能从过往成功和失败中学习。

### 2.2 关键技术细节
- **系统工作流**：用户请求 → supervisor 构建状态分类账（state ledger）→ 调用子智能体（coder 生成脚本、debugger 纠错、plugin manager 提供插件指导、Python 数据分析、QA 审核）。所有步骤保存为 Groovy/Python 脚本和文档。
- **容器构建**：基于 Docker，预安装 Fiji/ImageJ 2.16.0/1.54p 及 Top 30 非内置插件（MorphoLibJ、StarDist、TrackMate、Labkit 等）。预打包 Cellpose 模型权重，Conda 管理 Python 环境。启动时预热 jgo/Maven 缓存，避免运行时网络获取。通过 noVNC 暴露虚拟桌面，以非特权用户运行。
- **RAG 知识库内容**：包括 Bankhead《Introduction to Bioimage Analysis》、NEUBIAS 培训资源、ImageJ 用户指南、插件文档（MorphoLibJ、3D ImageJ Suite 等）、Jupyter notebook 合集（BioImageAnalysisNotebooks）。文档采用混合稀疏/稠密嵌入存储在 Qdrant 中，运行时通过 Reciprocal Rank Fusion（RRF）检索。
- **子智能体专用工具集**：包括插件 UI 检查、文件读取、Fiji Log 窗口解析等。与外部平台通过 MCP（Model Context Protocol）集成，如 napari 可视化。
- **质量保证（QA）代理**：自动审核每个项目是否符合社区发布的图像和图像分析检查清单（Schmied et al., 2024），涵盖工作流和图像发布两个部分，分 minimal/recommended/ideal 三级，输出 QA Checklist Report.md。

### 2.3 算法/流程（文字说明）
- 流程示例（寄生虫凋亡任务）：
  1. **刚性配准**：使用 TurboReg 将固定后图像的相位对比通道与活细胞电影最后一帧的相位对比通道对齐，并将变换应用于 TUNEL 和 DAPI 通道。
  2. **ROI 强度测量**：基于提供的寄生虫和背景 ROI，在活细胞最后一帧的绿色和红色通道测量平均强度，扣除背景平均后计算 R/G 比值，最终值为 10 - R/G。
  3. **细胞核分割与 TUNEL 分类**：基于 DAPI 分割细胞核，计算每个细胞核的 TUNEL 平均强度，使用 Otsu 阈值将其分为阳性/阴性。
  4. **寄生虫-细胞核关联**：将每个寄生虫 ROI 分配给最近（在最大距离内）的细胞核，否则标记为未分配。
  5. **统计与绘图**：合并结果表，比较 TUNEL+ 和 TUNEL- 组的 10 - R/G 值，使用 Mann-Whitney U 检验、Welch t 检验、Cliff's delta，并生成小提琴/箱线/散点图。

## 3. 实验设计

### 3.1 数据集与场景
- **案例一：小鼠肌肉干细胞的细胞追踪（UI 引导模式）**

- **案例二：寄生虫感染细胞的凋亡行为（自动多步工作流模式）**
  - 数据：来自 Baars et al. (2023) 的两个采集：一个活细胞电影（相位对比、mKikume 绿色/红色，多个时间点）和一个固定终点 TUNEL 染色图像（TUNEL、DAPI）。同时提供手工注释的 45 个寄生虫 ROI 及其背景 ROI。
  - 任务：自动对齐两幅图像、测量寄生虫荧光强度计算增殖指数（10 - R/G）、将寄生虫分配给宿主细胞、分类 TUNEL+/-、生成统计图和结果表。
  - 结果：智能体成功为 38/45 个寄生虫 ROI 分配了细胞核，并在 TUNEL+ 和 TUNEL- 组间重现了统计上相似的增殖指数差异（p=0.016 vs 专家 p=0.015），但灵敏度仅 50%（10 个专家标记的 TUNEL+ 中 5 个被检测到），特异性 100%。智能体自主识别并记录了这些不足。

- Agentic-J 将容器化多智能体系统与自然语言界面结合，使生物学家无需编程和软件配置即可完成复杂的 Fiji 图像分析工作流。

## 7. 优点

- **可复现性**：每个项目自动生成带文档的脚本、结果表和 QA 报告，方便分享和追溯。
- **可扩展性**：支持通过 MCP 协议集成外部工具（如 napari）；技能、插件、错误数据库均可增量更新。
- **教育价值**：知识库基于教育材料（教材、notebooks），智能体可提供参数解释和代码审查，有助于用户学习图像分析。
