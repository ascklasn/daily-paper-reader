## 摘要
生物医学代理有望自动化复杂的生物学工作流程，但当前系统面临两个根本瓶颈：生物信息学工具在接口和执行环境上高度异构，而代理规划仍依赖于基于平面提示的工具描述检索。随着生物医学软件生态系统的扩展，工具覆盖范围与上下文大小之间的耦合导致工具混淆、规划不稳定和执行效率低下。我们提出了BioManus，一种基于结构化生物学能力上的图脚手架规划构建的MCP原生生物医学代理。BioManus首先引入了BioinfoMCP编译器，将异构生物信息学软件转换为标准化的MCP服务器，从而产生一个大型可执行的MCP生态系统。然后，它将这个生态系统组织为一个关于工具、操作、数据类型和工作流程阶段的类型化异构MCP图。在推理时，BioManus检索紧凑的任务特定子图，合成操作级别的工作流程脚手架。这种设计将规划复杂性与原始工具库存规模解耦，在高召回率检索下实现了上下文压缩比Θ(N / (h * m_bar))，其中N是工具总数，h是工作流程时间范围，m_bar（远小于N）是每个操作的平均候选工具数量。在BioAgentBench和LAB-Bench上的实验表明，与先进的生物医学代理基线相比，BioManus在执行准确性、工作流程有效性和上下文效率方面均有提升。这项工作表明了一种范式转变：可扩展的生物医学推理需要结构化的可执行能力图，而不是越来越大的提示级工具检索。

## Abstract
Biomedical agents promise to automate complex biological workflows, yet current systems face two fundamental bottlenecks: bioinformatics tools are highly heterogeneous in interfaces and execution environments, while agent planning still relies on flat prompt-retrieved tool descriptions. As biomedical software ecosystems grow, this coupling between tool coverage and context size leads to tool confusion, unstable planning, and inefficient execution. We introduce BioManus, an MCP-native biomedical agent built on graph-scaffolded planning over structured biological capabilities. BioManus first introduces the BioinfoMCP Compiler, which converts heterogeneous bioinformatics software into standardized MCP servers, yielding a large executable MCP ecosystem. It then organizes this ecosystem as a typed heterogeneous MCP graph over tools, operations, datatypes, and workflow stages. At inference time, BioManus retrieves compact task-specific subgraphs, synthesizes operation-level workflow scaffolds. This design decouples planning complexity from raw tool inventory size, achieving a context compression ratio of Theta(N / (h * m_bar)) under high-recall retrieval, where N is the total tool count, h is the workflow horizon, and m_bar (much smaller than N) is the average number of candidate tools per operation. Experiments on BioAgentBench and LAB-Bench show that BioManus improves execution accuracy, workflow validity, and context efficiency over advanced biomedical agent baselines. This work suggests a paradigm shift: scalable biomedical reasoning requires structured executable capability graphs rather than increasingly larger prompt-level tool retrieval.

#### 2. 方法论：核心思想与关键技术细节

论文提出了 **BioManus**，一个基于 **MCP**（Model Context Protocol）原生架构和图脚手架规划（Graph-Scaffolded Planning）的生物医学代理。其核心思想是：**先构建结构化的“能力图”，再基于图进行规划，而非基于平面文本**。

- **核心组件一：BioinfoMCP编译器**
    - **功能**：将异构的生物信息学软件（CLI、库、容器）自动转换为标准化的 **MCP服务器**。
    - **流程**：三段式流程，包括（1）准备阶段：从手册、帮助文档等提取语义和执行规范；（2）执行阶段：使用LLM骨架生成MCP服务器代码，并进行迭代式语法和接口校验；（3）交付阶段：将服务器与依赖、Docker环境打包。
    - **成果**：构建了包含910个MCP服务器和3500个可调用工具的庞大生态系统，覆盖8个生物学领域。

- **核心组件二：类型化能力图（Typed Capability Graph）**
    - **功能**：将MCP生态系统组织成一个有向、异构图 `G = (V, E)`。
    - **节点（V）**：定义了六种节点类型，包括**工具节点**（VT）、**操作节点**（VO，抽象语义相同的工具）、**数据类型节点**（VD）、**能力节点**（VC）、**工作流阶段节点**（VS）和**MCP服务器节点**（VR）。
    - **边（E）**：定义了语义和执行关系，如 `implements`(操作→工具)， `consumes/produces`(数据类型↔操作)，`hosts`(服务器→工具)，`follows`(操作间的软工作流过渡)。
    - **意义**：该图显式地编码了生物学工作流的结构性知识，为规划提供了丰富的路线图。

- **核心组件三：图脚手架规划（Graph-Scaffolded Planning）**
    - **推理流程**：
        1.  **任务结构化**：将自然语言查询 `q` 转化为结构化任务说明（含生物实体、数据类型、目标）。
        2.  **图检索**：基于任务说明，通过类似GraphRAG的方式从全局图中检索出一个紧凑的“任务特定子图”，其中包含了候选操作路径和数据类型信息。
        3.  **操作级规划**：LLM规划器不再直接面对海量工具，而是首先在抽象的“操作”层面（如`quality_control`, `alignment`）进行推理，合成一个操作级执行路径 `P = (o1, o2, ..., oh)`。
        4.  **工具绑定**：将每个操作 `oi` 绑定到其对应的少数候选工具集 `T(oi)` 上。
        5.  **动态注册**：仅将涉及到的MCP服务器动态注册到执行环境，避免了全局上下文的污染。

- **复杂性分析（核心优势）**：
    - **传统方法**：`C_prompt = O(k * ℓt)`，随检索工具数 `k`（接近N）线性增长。
    - **BioManus**：`C_graph = O(h * (ℓo + m_bar * ℓs))`，其中 `h` 是工作流步骤数，`m_bar` 是每步的候选工具平均数（`m_bar << N`）。
    - **结论**：BioManus将规划复杂度从与工具库存总量 `N` 解耦，转变为仅与任务本身的复杂性 `h` 和 `m_bar` 相关，实现了理论上的上下文压缩比 `Θ(N / (h * m_bar))`。
