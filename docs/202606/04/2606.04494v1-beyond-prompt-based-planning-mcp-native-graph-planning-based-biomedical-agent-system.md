## 摘要
生物医学智能体有望自动化复杂生物工作流程，但当前系统面临两个基本瓶颈：生物信息学工具在接口和执行环境上高度异构，而智能体规划仍依赖于扁平化的提示检索工具描述。随着生物医学软件生态系统的扩展，工具覆盖范围与上下文规模之间的耦合导致工具混淆、规划不稳定和执行效率低下。我们提出BioManus，这是一个基于MCP原生的生物医学智能体，建立在结构化生物能力之上的图支架规划。BioManus首先引入BioinfoMCP编译器，将异构生物信息学软件转换为标准化的MCP服务器，形成可执行的大型MCP生态系统。然后，它将此生态系统组织为工具、操作、数据类型和工作流阶段上的类型化异构MCP图。在推理时，BioManus检索紧凑的任务特定子图，合成操作级工作流支架。这种设计将规划复杂度与原始工具库存规模解耦，在高召回检索下实现上下文压缩比Θ(N / (h * m_bar))，其中N是工具总数，h是工作流步数，m_bar（远小于N）是每个操作的平均候选工具数。在BioAgentBench和LAB-Bench上的实验表明，与先进的生物医学智能体基线相比，BioManus在执行准确性、工作流有效性和上下文效率方面均有提升。这项工作表明一个范式转变：可扩展的生物医学推理需要结构化的可执行能力图，而非越来越大的提示级工具检索。

## Abstract
Biomedical agents promise to automate complex biological workflows, yet current systems face two fundamental bottlenecks: bioinformatics tools are highly heterogeneous in interfaces and execution environments, while agent planning still relies on flat prompt-retrieved tool descriptions. As biomedical software ecosystems grow, this coupling between tool coverage and context size leads to tool confusion, unstable planning, and inefficient execution. We introduce BioManus, an MCP-native biomedical agent built on graph-scaffolded planning over structured biological capabilities. BioManus first introduces the BioinfoMCP Compiler, which converts heterogeneous bioinformatics software into standardized MCP servers, yielding a large executable MCP ecosystem. It then organizes this ecosystem as a typed heterogeneous MCP graph over tools, operations, datatypes, and workflow stages. At inference time, BioManus retrieves compact task-specific subgraphs, synthesizes operation-level workflow scaffolds. This design decouples planning complexity from raw tool inventory size, achieving a context compression ratio of Theta(N / (h * m_bar)) under high-recall retrieval, where N is the total tool count, h is the workflow horizon, and m_bar (much smaller than N) is the average number of candidate tools per operation. Experiments on BioAgentBench and LAB-Bench show that BioManus improves execution accuracy, workflow validity, and context efficiency over advanced biomedical agent baselines. This work suggests a paradigm shift: scalable biomedical reasoning requires structured executable capability graphs rather than increasingly larger prompt-level tool retrieval.

### 2. 论文提出的方法论

**核心思想：** 构建一个结构化的生物能力图（Typed Capability Graph），让智能体在推理时检索紧凑的任务相关子图，并在此子图上进行工作流规划和工具绑定，从而将规划复杂度与全局工具库存大小解耦。

**关键技术细节与流程（BioManus系统）：**

1.  **BioinfoMCP编译器：工具标准化**
    *   **自动化转换**：将异构的生物信息学软件（CLI、Python、R包等）自动转换为标准化的MCP（Model Context Protocol）服务器。MCP提供了统一的、可编程的接口，解决了工具异构性问题。
    *   **三阶段流程**：准备阶段（提取文档和CLI规范）-> 执行阶段（LLM生成MCP服务器代码，并进行自动语法和接口校验，迭代修正直到成功）-> 交付阶段（打包依赖和Docker环境，确保可移植性和可重复性）。
    *   **产出**：一个包含910个MCP服务器和3500个可调用工具的、跨8个生物领域的大型MCP生态系统。

2.  **类型化能力图（Typed Capability Graph）构建**
    *   **异构图结构**：将MCP生态系统组织为一个有类型的异构图 G = (V, E)。
    *   **节点类型**：
        *   工具节点 (V_T)：具体的可执行MCP工具。
        *   操作节点 (V_O)：对语义相似工具的抽象（如“比对”、“质量控制”）。
        *   数据类型节点 (V_D)：如FASTQ、BAM、VCF等。
        *   能力节点 (V_C)：更高层次的原子能力，如“变异检测”。
        *   工作流阶段节点 (V_S)：如预处理、核心分析、报告生成。
        *   MCP服务器节点 (V_R)：托管具体工具的服务器。
    *   **边类型**：包括 `implements`（操作->工具）、`consumes/produces`（数据类型<->操作）、`hosts`（服务器->工具）、`typed_flow`（标准化数据流）等。
    *   **构建方式**：基于生成的MCP生态，通过LLM自动解析工具模式、自然语言描述，推断输入/输出数据类型、语义操作类别和工作流阶段。

3.  **图支架工作流规划（Graph-Scaffolded Planning）**
    *   **结构化查询**：将用户的自然语言查询转换为包含生物实体、能力、数据类型等语义锚点的结构化任务说明。
    *   **图检索**：利用类似GraphRAG的技术，从全局能力图中检索出一个紧凑的、任务特定的子图。检索是在操作级别（V_O）进行的，而非原始工具级别（V_T）。
    *   **规划与绑定**：LLM基于检索到的子图，首先规划一个操作级的执行路径 `P = (o1, o2, ..., oh)`。然后，将路径中的每个操作 `oi` 绑定到其关联的、有限的候选工具集 `T(oi)` 中的具体工具上。
    *   **动态执行**：只有选中的MCP服务器才会被动态注册并执行。

4.  **复杂度分析（公式推导）**
    *   **传统范式（Prompt-based）**：
        *   C_prompt ≈ C_query + k * l_t + C_instruction，其中k是检索工具数，l_t是工具描述平均长度。
        *   由于在高召回下 `k = Θ(N)`（N是工具总数），因此 `C_prompt = O(N * l_t)`，与N线性相关。
    *   **本文方法（Graph-scaffolded）**：
        *   C_graph ≈ h * (l_o + m̄ * l_s)，其中h是工作流步数（通常很小），l_o是操作描述长度，m̄是每个操作的平均候选工具数（通常m̄ << N），l_s是工具模式长度。
        *   因此，规划复杂度 `C_graph` 由**工作流复杂度（h）**决定，而不是工具总数 **N**。上下文压缩比达到 `Θ(N / (h * m̄))`，实现了与总工具数目的解耦。

### 7. 优点

*   **理论清晰，问题精准**：对“提示检索”范式的瓶颈诊断非常精准，并用复杂度分析和实证数据（图2）有力地支撑了其核心论点。
*   **方法创新，结构优雅**：提出的图支架规划方法（Graph-Scaffolded Planning）概念清晰，将结构的先验知识（能力图）嵌入到规划过程中，优雅地解决了可扩展性问题。
*   **系统完整，实用性强**：BioManus不仅仅是一个概念，而是包含了从工具编译（BioinfoMCP Compiler）到图谱构建再到动态执行的完整系统，具有很强的实用价值。将异构工具标准化为MCP服务器的思路，解决了低层次执行碎片化问题。
*   **实验严谨，说服力强**：实验设计针对核心论点（可扩展性）进行了专门设计（工具规模递增的消融），实验公平性控制较好（统一后端），对比基线全面。
*   **泛化性好**：MCP编译器在不同LLM后端（Gemini, GPT, Kimi）上都展现了稳定性，表明其方法的鲁棒性。

### 8. 不足与局限

1.  **依赖文档质量**：BioinfoMCP编译器的可靠性依赖于现有工具文档（如手册、帮助信息）的质量。文档不全或不规范时，生成的MCP服务器代码可能不稳定。
2.  **图谱构建自动化程度有限**：当前的类型化能力图依赖于LLM进行语义标注和关系推断，其质量受到LLM能力限制。未来可融入更精细的生物本体（Ontology）和专家人工校验的流程知识。
3.  **检索质量是关键**：图规划的有效性在很大程度上依赖于图检索的质量。如果未检索到关键的子图，可能会导致规划失败或路径次优。论文提到这是一个局限性。
4.  **基准规模有限**：BioAgentBench只有10个任务，评估的广度和深度有限。虽然LAB-Bench提供了更多场景，但整体基准仍然不足以完全衡量系统在极端复杂且长序列生物工作流上的能力。
5.  **缺乏与其它框架的对比**：实验没有与同样强调结构化的、非MCP的现代生物信息学工作流系统（如Snakemake, Nextflow）进行对比，尽管这些系统本身不具备端到端的智能体规划能力。
6.  **仅关注文本/命令行工具**：系统目前主要处理文本参数的命令行和API工具，对于复杂的、基于图形用户界面的软件或需要物理实验操作的任务，尚未覆盖。
7.  **伦理与安全**：作为辅助系统，论文明确指出了专家监督的必要性，但并未深入讨论智能体在自动化执行时可能出现的安全风险（如误操作导致数据损坏）或伦理问题（如生物实验的自动化决策责任）。
