## 摘要
随着大型语言模型（LLM）在医疗场景中的日益部署，对生成或现有文本中的错误进行准确检测和纠正变得至关重要，因为即使是微小的错误也可能对患者安全构成风险。现有的错误检测与纠正方法（包括自动检查和基于启发式的方法）在未见过的数据集上泛化能力不强。在本文中，我们提出MedGuards作为一种医疗安全护栏，这是一个新的框架，将医疗错误检测与纠正视为多智能体上下文学习任务。专门的智能体分别检测、定位和纠正错误，而置信度引导的仲裁机制利用推理轨迹和置信度分数来解决分歧。该设计增强了可解释性、鲁棒性和适应性，且无需对基础LLM进行额外训练。此外，我们引入了关键词优先纠正分数（KPCS），这是一种新的评估指标，考虑了参考文本中的关键关键词是否被正确生成，从而提供比传统指标更全面的评估。在包含临床笔记的四个多语言医疗数据集上的实验表明，所提框架在多个指标和模型上均有显著改进。我们的目标是使LLM在现实医疗应用中的部署更加安全。为实现可重复性，我们在https://github.com/congboma/MedGuards上公开了代码。

## Abstract
As Large Language Models (LLMs) are increasingly deployed in healthcare settings, accurate error detection and correction in generated or existing text becomes critical, as even minor mistakes can pose risks to patient safety. Existing methods for error detection and correction, including automated checks and heuristic-based approaches, do not generalize well across unseen datasets. In this paper, we propose MedGuards as a medical safety guardrail, which is a new framework that treats medical error detection and correction as a multi-agent in-context learning task. Specialized agents separately detect, localize, and correct errors, while a confidence-guided arbitration mechanism resolves disagreements using reasoning traces and confidence scores. This design enhances interpretability, robustness, and adaptability, without requiring additional training of the base LLMs. Additionally, we introduce the Keyword-Prioritized Correction Score (KPCS), a new evaluation metric that considers whether critical keywords within the reference text are generated correctly, providing a more comprehensive assessment than conventional metrics. Experiments across four multilingual medical datasets consisting of clinical notes demonstrate significant improvements by the proposed framework across several metrics and models. Our aim is to enable safer deployment of LLMs in real-world healthcare applications. For reproducibility, we make our code publicly available at https://github.com/congboma/MedGuards.

-   **研究动机**：
    1.  错误纠正过程本身包含检测、定位和纠正的逐步推理，这与思维链（Chain-of-Thought, CoT）的思路高度契合。
    2.  医疗应用对高可靠性要求极高，依赖单一模型在分布外场景下容易出现不确定性、偏差或失败。多智能体系统可以通过聚合多个模型的预测来提高鲁棒性。
-   **研究含义**：旨在构建一个无需额外训练且可解释的、高效的“医疗安全护栏”，提升LLM在临床应用中生成内容的可靠性，为安全部署提供系统性解决方案。

#### 2. 方法论：核心思想、关键技术细节

-   **核心思想**：将医疗错误检测与纠正框架化为一个**多智能体上下文学习（Multi-Agent In-Context Learning, MA-ICL）** 任务。整个流程被拆解为三个子任务，由专门的智能体协作完成，并通过仲裁机制解决分歧。

-   **关键技术细节**：

    1.  **任务分解（基于CoT）**：将医疗错误纠正任务分解为三个连续的步骤：
        -   **错误检测（Error Detection）**：判断临床笔记中是否存在错误。
        -   **错误定位（Error Localization）**：识别包含错误的句子。
        -   **错误纠正（Error Correction）**：生成纠正后的句子。

    2.  **多智能体自一致性（Multi-Agent Self-Consistency）**：
        -   在检测和定位阶段，分别由两个独立的智能体（`Agent_1`和`Agent_2`）处理同一输入。
        -   如果两者意见一致，直接输出结果；如果存在分歧，则引入第三个仲裁智能体（`Agent_3`）进行裁决。

    3.  **上下文学习（ICL）仲裁（ICL-based Arbitration）**：
        -   仲裁智能体不依赖参数更新，而是将两个初始智能体的输出（包括其推理过程`r`和置信度分数`c`）作为上下文输入，综合判断并做出最终决定。这使得仲裁过程本身就是一次ICL学习任务。

    4.  **字符串对齐**：在定位阶段，使用Python的`SequenceMatcher`，通过字符级别的长公共子序列（LCS）相似度，将模型预测的错误句子自动对齐到原始临床笔记中。这对于处理模型可能产生的微小词汇变化或幻觉非常有效。

    5.  **新的评估指标：关键词优先纠正分数（Keyword-Prioritized Correction Score, KPCS）**：
        -   旨在解决传统指标忽略关键临床实体重要性的问题。
        -   **计算步骤**：
            1.  **关键词检查**：检查纠正后的句子`Sc`是否包含参考句子`Sr`中的预定义关键词（如诊断、药物等），计算关键词覆盖率`K`。
            2.  **语义和词汇相似度**：计算`Sc`和`Sr`在ROUGE-1、BERTScore和BLEURT三个指标上的平均值`M`。
            3.  **加权融合**：最终KPCS = α*K + (1-α)*M。α是一个平衡参数（文中设为0.5），用于权衡关键词精确度和整体语义流畅度。

-   **算法流程**：
    输入临床笔记x -> 两个检测智能体（若结果一致则直接输出，否则仲裁） -> 获得是否存在错误的判断d。
    若存在错误（d=1）-> 两个定位智能体（若结果一致则直接输出，否则仲裁） -> 获得错误句子s。
    -> 一个纠正智能体基于原文本和错误句子生成纠正版本ŝ。

#### 7. 优点

-   **方法设计新颖**：首次将多智能体ICL框架系统性地应用于医疗错误检测与纠正任务，将任务分解、自一致性和上下文仲裁巧妙结合。
-   **即插即用**：无需对LLM进行微调，可直接应用于现有模型，部署成本低，适应性强。
-   **高度可解释**：每个智能体都输出其推理过程和置信度，仲裁过程也基于这些信息，使整个错误检测与纠正流程透明、可追溯。
-   **评估指标创新**：提出的KPCS指标聚焦于临床关键实体，弥补了传统指标的不足，评估结果更具临床相关性。
-   **实验设计严谨全面**：验证了框架的鲁棒性和有效性，涵盖了多数据集、多语言、详细的消融分析和人类评估，说服力强。

#### 8. 不足与局限

-   **计算开销**：多智能体推理需要多次调用LLM API，带来了更高的延迟和成本。虽然论文承认这一点，但未提出有效的优化方案（如并行处理），这可能限制了其在延迟敏感场景下的直接应用。
-   **KPCS指标的局限**：KPCS的关键词部分目前依赖精确的字符串匹配，无法处理医学同义词或缩写。论文也承认未来需要集成医学知识库来改进这一点。
-   **对基础模型的依赖**：MedGuards的性能高度依赖于所选用的LLM骨干网络。在较弱模型上，虽然有提升，但绝对性能仍有限。框架本身并未解决LLM固有可能存在的知识缺陷或偏见。
-   **错误类型的覆盖**：MEDEC和MedErrBench数据集覆盖的错误类型相对有限（如10-15种），可能无法完全反映真实临床笔记中所有复杂的、非结构化的错误。
-   **未进行真实临床环境验证**：论文的实验均基于公开的、人为构造或从考试题衍生的数据集，缺乏在真实电子健康记录（EHR）或临床工作流中进行的前瞻性验证。其在实际临床部署中的表现、医生接受度和合规性有待进一步研究。
-   **英文框架的局限**：虽然验证了多语言能力，但提示词（Prompt）和核心设计都是基于英文的，其在其他语言上的优化和鲁棒性有待进一步挖掘。
