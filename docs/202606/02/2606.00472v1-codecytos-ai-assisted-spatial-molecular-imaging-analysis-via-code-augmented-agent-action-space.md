## 摘要
传统的组织图像分析软件为细胞分析提供了基础功能，包括分割、基本形态特征提取和空间组织分析。然而，这些工具通常需要人工干预，并且与代码驱动的自动化集成度不高，限制了复杂空间组织研究的效率和可扩展性。此外，它们对自定义分析的灵活性有限，通常仅支持一组固定的预实现空间细胞特征。为解决这些限制，我们提出CodeCytos，一种基于编码的推理智能体框架，能够实现与空间分子成像数据的动态、可编程交互，以提高自动化和定制化能力。CodeCytos旨在简化自定义空间细胞特征的探索，并适应多样化的研究需求。我们通过四个来自不同组织类型（额叶皮层、非小细胞肺癌、胰腺和扁桃体）的专家精选数据集进行案例研究，展示了其实用性。我们在现实的最小提示设置下评估CodeCytos，即生物科学家提出简单问题，但不提供任务特定指令或关于空间细胞分析的上下文信息，并基准测试了多个具有强编码能力的大语言模型骨干。我们进一步证明，引入定制的、领域无关的少样本上下文编码推理示例（随机采样的空间分析领域之外的演示）可以显著提升性能，而无需昂贵、专家精心制作的领域内演示。总体而言，CodeCytos优于基线方法，突出了代码动作智能体在辅助空间分子成像中自定义特征探索和加速生物标志物发现方面的潜力。

## Abstract
Conventional tissue image analysis software provides foundational capabilities for cellular analysis, including segmentation, basic morphological feature extraction, and spatial organization analysis. However, these tools often require manual intervention and are not well integrated with code-driven automation, limiting efficiency and scalability for complex spatial tissue studies. In addition, they offer limited flexibility for custom analyses, as they typically support only a fixed set of pre-implemented spatial cellular features. To address these limitations, we propose CodeCytos, a coding-based reasoning agent framework that enables dynamic, programmable interaction with spatial molecular imaging data to improve automation and customization. CodeCytos is designed to streamline the exploration of custom spatial cellular features and adapt to diverse research needs. We demonstrate its utility through case studies on four expert-curated datasets from distinct tissue types: frontal cortex, non-small-cell lung cancer, pancreas, and tonsil. We evaluate CodeCytos under a realistic minimal prompt setting, where bioscientists pose simple questions without task-specific instructions or contextual information about spatial cellular analysis, and benchmark multiple LLM backbones with strong coding capabilities. We further show that incorporating tailored, domain-agnostic few-shot in-context coding-reasoning examples (randomly sampled demonstrations outside the spatial analysis domain) can substantially improve performance without requiring costly, expert-crafted in-domain demonstrations. Overall, CodeCytos outperforms baseline approaches, highlighting the potential of code-action agents to assist with custom feature exploration in spatial molecular imaging and to accelerate biomarker discovery.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：基于 **ReAct（推理与行动协同）** 和 **CodeAct（代码作为行动）** 两种范式的融合。
    1.  **ReAct 模式**：智能体在解决任务时，进行多轮“**思考（Thought）** → **行动（Action）** → **观察（Observation）**”的循环。思考步骤让LLM进行推理和规划，行动步骤则与环境交互。
    2.  **CodeAct 扩展**：将 ReAct 中的“行动”具体化为**可执行的 Python 代码**。这利用了LLM强大的代码生成能力，将智能体的动作空间从受限的自然语言或预定义工具，扩展到无限的可能执行代码，极大地提升了灵活性和自动化程度。
- **关键技术细节与算法流程**：
    1.  **智能体交互循环**：给定一个生物学家的问题（如“T细胞在30微米网格中的方差均值比是多少？”），CodeCytos 启动一个 **Thought-Code-Observation** 迭代循环：
        - **Thought (思考)**：智能体分析当前上下文（包括用户问题、之前步骤的代码和输出、错误信息等），制定下一步计划。例如，它会想：“首先，我需要从细胞列表中滤出所有T细胞，然后获取其多边形坐标并计算中心点...”。
        - **Code (生成与执行代码)**：智能体根据思考结果生成具体的 Python 代码，并调用沙盒环境执行。代码会使用标准库（如 numpy, scipy, shapely, networkx等）来完成具体计算。
        - **Observation (观察)**：执行代码后，智能体会获取输出结果、日志或任何错误信息。这些观察结果被追加到上下文中，用于指导下一步的思考。如果代码出错，智能体会利用错误信息进行自我修正和重新执行。
    2.  **少样本编码-推理提示（Few-shot Coding-Reasoning Prompting）**：
        - **问题**：虽然零样本CodeAct表现不错，但开源模型与闭源模型间存在巨大性能差距。
        - **解决方案**：在系统提示中预置少量（如2-3个）**领域无关（Domain-Agnostic）** 的编码-推理示例。这些示例**并非**来自空间分析领域，而是通用的、展示正确迭代格式（Thought/Code/Observation标签）和编程思维模式的例子。
        - **目的**：这些示例的作用不是提供领域知识，而是**规范智能体的行为模式**，引导其生成符合格式要求、具备逐步推理和代码调试能力的输出。这种方法避免了需要昂贵的、专家精心制作的领域内演示。
- **公式表示**：
    - 智能体的上下文：$c_t = (o_1, a_1, ..., o_{t-1}, a_{t-1}, o_t)$
    - 混合动作空间：$\tilde{A} = A \cup L$，其中 $A$ 是可执行代码（CodeAct），$L$ 是自然语言思考（Thought）。
    - 智能体策略：$a_t \sim p_\theta(\cdot | \text{[系统提示 + 少样本示例]}, c_t)$。其中 $a_t$ 包含思考 $r_t$ 和代码 $z_t$。

- **方法创新性强**：巧妙地将ReAct的推理能力和CodeAct的代码执行能力相结合，创建了一个高度灵活和自动化的智能分析系统，直接解决了传统软件灵活性差的核心痛点。
- **领域无关的少样本示例**：这是**本文最核心的亮点**。它颠覆了传统上需要精心制作领域内示例的观念，证明了只规范智能体*如何思考和执行代码*的行为模式，就能在特定任务上取得巨大成功。这极大地降低了应用门槛和成本。
- **实验设计严谨**：采用最小提示设置，最大程度避免了人为偏差。基准测试集由专家策划，覆盖多组织、多特征类别，确保了评估的全面性和客观性。
- **分析深入**：
    - 引入 **AUP@k** 作为聚合指标，便于比较不同方法在多次尝试下的整体表现。
    - 清晰的消融实验（零/一/少样本）和按特征类别分析，有效揭示了各组件和方法的性能特点及瓶颈。
    - 对模型表现进行了聚类分析，揭示了不同LLM骨干的性能模式。
- **实践价值高**：通过大量交互示例展示了框架的现实应用场景，能够直接服务于生物学家，有望加速生物标志物发现。论文承诺开源代码和基准，有利于该领域的进一步发展。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制

- **计算成本**：论文未量化智能体运行的计算成本。虽然提示工程成本降低了，但运行一个强大的LLM（123B参数）进行多轮迭代推理和代码执行，其**推理成本**（时间、金钱、能源）可能非常高，这对于资源有限的实验室可能是不小的负担。
- **LLM依赖**：系统的性能和可靠性高度依赖于底层LLM的编程能力和推理能力。如果LLM能力不足（如生成代码有语法错误、逻辑错误或不正确调用库），智能体将失败。论文也表明，即使多次尝试（Pass@k），仍存在一些无法解决的难题。
- **基准规模的局限**：虽然548个问题已经很丰富，但相对于整个空间生物学领域的复杂性，基准仍相对有限。未来的工作需要在更广泛的组织类型和更复杂的分析任务上验证。
- **评估指标的局限**：
    - **输出匹配**：评估指标（成功率和Pass@k）依赖于最终答案与“ground-truth”的严格匹配。对于绝对值比较类问题（如均值、方差等），匹配是清晰的。但对于更模糊的判断或图形结果，此评估方法可能不够精确。
    - **忽略中间步骤质量**：成功指标只关注最终答案，忽略了中间推理过程和代码质量。一个得到正确结果的“运气好”的智能体与一个逻辑严谨的智能体在指标上表现相同。
- **流程限制**：
    - **智能体步数限制**：实验中将每条智能体轨迹限制在最多10步。更复杂的任务可能需要更多步骤，这可能导致任务过早失败。
    - **不适用于所有任务**：论文承认CodeCytos并非要取代现有软件，而是作为一种补充，适用于现有工具不支持的自定义分析场景。
    - **M3ToolEval 上的增益较小**：虽然领域无关示例有效，但在一些主干模型（如Hermes-4-70B, GLM-4.5-Air）上效果不明显甚至下降，且整体增益小于在空间分析基准上的增益。这表明其效果可能因模型和任务类型而异。
- **用户交互成本**：虽然用户只需提出自然语言问题，但流程中（如图1所示）生物学家仍需要“选择细胞分割和分类工具”，这仍需要一定的专业知识。并非完全“傻瓜式”操作。
