## 摘要
历史医学档案和传统药物在药物发现方面具有巨大潜力，并且仍然是当前药物开发的主要来源。然而，前本体论的散文和特殊的分类法阻碍了数据的标准化和医学现代化，使其无法用于当前的生物医学流程。此外，现有的LLM代理系统，无论是工具调用、检索增强还是代理深度研究，都无法大规模地将此类文本转化为可验证的药物发现线索。我们通过DeepRoot弥补了这一差距，这是一个多智能体LLM系统，它共同构建并利用一个经过验证的知识图谱，表明接地和推理——常常被混为一谈——是系统可以组合用于治疗推理的可分离轴。应用于《神农本草经》，DeepRoot在R@20下恢复了21个保留的化合物-疾病治疗对中的10个（47.6%对比原始语料LLM的4.8%和约2.4%的随机），并且在推理质量上以LLM作为评判的审核中优于基线LLM以及直接通过工具调用访问DeepRoot本身查询的相同API的LLM。使用工具的LLM在87%的声明上产生幻觉证据，而DeepRoot为7-10%。仅图谱推理的幻觉率为0%，但推理连贯性得分最低；DeepRoot KG+LLM是唯一在两个轴上均获胜的条件，为历史医学知识的系统挖掘和再利用指明了一条道路。

## Abstract
Historical medical archives and traditional medicines hold immense potential for drug discovery and remain a primary source for current drug development. However, pre-ontological prose and idiosyncratic taxonomies prevent the standardization and medical modernization of the data for use in current biomedical pipelines. Furthermore, no existing LLM agent system, whether tool-calling, retrieval-augmented, or agentic deep-research, can convert such text into verifiable drug-discovery leads at scale. We close this gap with DeepRoot, a multi-agent LLM system that jointly builds and utilizes a verified knowledge graph, showing that grounding and reasoning -- often conflated -- are separable axes the system can compose for therapeutic reasoning. Applied to the Shen Nong Ben Cao Jing, DeepRoot recovers $10$ of $21$ held-out compound-disease treatment pairs at R@$20$ ($47.6\%$ vs $4.8\%$ for a raw corpus LLM and $\sim\!2.4\%$ random) and dominates an LLM-as-judge audit for reasoning quality over baseline LLMs and LLMs with direct tool-call access to the same APIs DeepRoot itself queries. Tool-using LLMs hallucinate evidence on $87\%$ of claims, versus 7-10% for DeepRoot. Graph-only inference hallucinates $0\%$ but ranks lowest on reasoning coherence; DeepRoot KG+LLM is the only condition to win on both axes, pointing toward a route for systematic mining and repurposing of historical medical knowledge.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：历史医学文本（如《神农本草经》等传统药物文献）蕴含巨大的药物发现潜力（例如青蒿素、吗啡的发现），但其语言是**前本体论散文**，分类法特异，阻碍了标准化和现代化利用。现有 LLM 系统（包括工具调用、检索增强、深度研究代理）都无法大规模、可靠地将这类文本转换为可验证的药物发现线索。
- **整体含义**：论文提出 LLM 的**事实依据（grounding）** 与**推理（reasoning）** 是可分离的两个维度，通过多智能体系统先联合构建验证过的**知识图谱（KG）**，再基于图谱进行机制推理，可以有效挖掘历史医学知识，并实现低幻觉、高可审计性的治疗候选发现。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将**接地（KG 构建）**与**推理（KG 上的路径评分 + LLM 批判）**分离。先由多个专门代理协同构建严格验证的 KG（Neo4j），再由发现代理和批评代理利用图结构进行闭环机制推理。
- **系统架构**：DeepRoot 包含两个阶段：
  - **DeepRoot Assembly（组装阶段）**——7 个代理按依赖顺序执行：
    1. **提取代理**：从原始文本块中提取实体（Source、Malady、Preparation）和关系（TREATS_TRADITIONALLY、PREPARED_AS）。
    2. **审计代理**：规范化源名称、合并重复实体、验证证据跨度的子串匹配（防止 LLM 幻觉的跨步）。
    3. **疾病映射代理**：“生成-验证”协议：LLM 提出规范疾病名称，然后通过**容错精确匹配**从 ICD-10/MeSH/SNOMED 等本体服务验证编码（绝不信任 LLM 生成的代码）。
    4. **源→化合物连接器**：通过 COCONUT 2.0（自然产物）和 PubChem（化学品）将生物源链接到化合物，化合物身份由 RDKit 计算的 InChIKey 保证。
    5. **化合物→靶点连接器**：通过 ChEMBL 查询，以 pChEMBL≥5 过滤活性，支持单蛋白、蛋白复合体、蛋白家族、病原体四种靶点类型。
    6. **靶点→疾病连接器**：根据靶点类型分发：单蛋白走 Open Targets GraphQL，蛋白复合体做亚基扇出，病原体走 OLS4 本体遍历 + LLM 安全网。
    7. **审查代理**：两遍归档（第一遍规则性捕获 OCR 伪影等，第二遍 LLM 批处理模糊节点），软删除。
  - **DeepRoot Discovery（发现阶段）**：
    - **批评代理**：接收 KG 聚合的路径信息（桶层级、闭环计数等）和结构化 payload，输出四层判决（VALIDATED/PLAUSIBLE/WEAK/UNSUPPORTED）、多维分数、关键证据、关注点列表。提示要求引用特定输入字段，不猜测。
    - **发现（提名）代理**：完全确定性（无 LLM），给定目标现代疾病，向后遍历 KG 枚举候选化合物，再向前评分，按 (闭环标志, 前向桶层级, 源数量, 疾病数量, 分数) 排序。
- **关键技术细节**：
  - **路径评分**：每条边有证据类型标签（如 chembl_mechanism, ot_association_strong），映射到四个桶 T1>T2>T3>T4；路径桶 = 最弱边桶（最小值），路径分数 = 边置信度乘积。先以桶排序，同桶内以分数排序。
  - **置信度先验**：平坦（而非学习），基于生物医学表面效度设定（例如 T1 桶 0.70-0.95，T4 桶 0.40-0.50）。原始外部分数（pChEMBL、OT 总体分）保留为边属性供后续重新校准。
  - **幂等写入**：所有写操作为 MERGE（基于 InChIKey/ChEMBL ID/规范疾病名），支持断点续跑。

## 3. 实验设计

### 数据集
- **语料**：《神农本草经》，分为 71 个文本块，共涉及 415 个源、294 种传统疾病、129 种现代疾病、18,012 种化合物、2,211 个靶点。
- **外部接地来源**：COCONUT 2.0、PubChem、ChEMBL、Open Targets、NCBI Taxonomy、OLS4、ICD-10、MeSH、SNOMED、MONDO、DOID。

### 对比方法
1. **原始语料 LLM**：仅给文本段落，要求其排名化合物候选。
2. **工具调用 LLM**：直接访问与 DeepRoot 相同 API（ChEMBL、Open Targets、PubMed、MeSH）。
3. **仅图谱推理**：仅使用确定性 Pass-1 信号，无 LLM。
4. **Biomni**（另一个生物医学推理代理环境）。
5. **多个 LLM 骨干**：Gemini 3.1 Pro / 2.5 Flash / 3.1 Flash Lite 三种配置。

### 实验场景
1. **知识图谱消融（边扰动）**（30 个闭环保留 claim，6 个扰动水平 0%~100%）：验证批评代理自信度随图谱完整性下降而下降，与原始 LLM 基线趋于一致。
2. **图谱引导的候选恢复（迷你语料库）**（30 个迷你语料库，每个含 3 个闭环源 + 7 个干扰源）：测量源召回@3、化合物召回@10、平均自信度。
3. **盲法恢复保留的已验证治疗对**（21 对符合历史可达性的 ChEMBL 适应症指示对，隐藏 KNOWN TREATS 边后重排）：比较 R@1/5/10/20、MRR。
4. **LLM 作为评判的推理质量评估**（30 个分层采样的 claim，7 种条件，由 Claude Sonnet 4.6 评估 6 个维度 + 幻觉率）：证据保真度、判决一致性、推理连贯性、临床映射、不确定性校准、可操作性。
5. **人类专家评估定性案例**（50 对源-疾病对），与 Biomni 比较疾病映射一致性和判决一致性。

  - **充分性**：覆盖了消融（验证 KG 贡献）、与多种基线对比（包括工具调用 LLM、仅图谱、Biomni）、盲法设置（隐藏真值）和人类专家评估。实验设计较完整。
  - **公平性**：工具调用 LLM 被设定为直接访问相同 API，但结果显示幻觉率 87%——作者指出预构建 KG 比推理时查询更能抑制幻觉。与 Biomni 的对比存在局限（Biomni 可访问最新论文，超出评判模型知识）。总体实验设计合理，但统计严谨性有提升空间（21 对保留无置信区间、单语料库泛化性未验证）。

## 6. 主要结论与发现

1. **KG 构建抑制幻觉**：DeepRoot 幻觉率 7-10%，远低于工具调用 LLM 的 87% 和文本+LLM 的 13%。仅图谱推理幻觉率 0% 但推理连贯性最差。
2. **KG+LLM 组合最优**：在所有 6 个推理维度上，DeepRoot（任何 LLM 骨干）均优于其它条件，是唯一在准确性和推理质量上同时获胜的方法。
3. **盲法恢复显著**：DeepRoot 在 R@20 恢复 47.6% 的保留验证对（10/21），远高于 LLM 基线 4.8% 和随机 2.4%。
4. **图谱结构有用性**：边扰动实验显示批评代理自信度随图谱完整性下降而单调下降，证明 KG 对 raisoning 有实际贡献。
5. **低成本可扩展**：图谱构建仅需 $0.25，且支持增量更新。

## 7. 优点

- **创新性**：将 grounding（事实验证）与 reasoning（图谱推理 + LLM 批判）**明确分离并组合**，展示了知识图谱在历史医学文本挖掘中的关键作用。
- **可审计性**：
  - 所有实体链接均经过确定性数据库验证（容错精确匹配），杜绝 LLM 幻觉本体编码。
  - 路径评分机制透明可解释（桶层级 + 边证据类型标签）。
  - 批评代理输出要求引用特定 payload 字段，支持溯源。
- **低成本高效率**：全文构建仅 $0.25，30-40 分钟完成，远低于推理时反复调用 API 的成本。
- **跨语料潜力**：系统设计不依赖特定语言或文本结构，可迁移至阿育吠陀等其它历史医学体系。
- **实验设计严谨**：涵盖消融、盲法、多维度 LLM 评判、人类评估，且公开代码和数据（GitHub）。

## 8. 不足与局限

- **语料局限**：仅使用单一语料库（《神农本草经》71 个块），未验证迁移性至其他历史文献（如阿育吠陀、西方草药志）。
- **样本量小**：保留测试集仅 21 对（单种子，无置信区间），统计功效有限。
- **置信度先验未校准**：路径评分使用的先验基于表面效度（flat priors），未从数据学习或校准。
- **疾病覆盖不全**：Open Targets 仅覆盖人类疾病，非现代适应症（如传统“毒”的广义概念）无法评分。
- **LLM 依赖**：DeepRoot Discovery 的推理质量受限于底层 LLM（Gemini）能力；批评代理的合理性也受 LLM 表现约束。
- **对比评估局限**：与 Biomni 的对比中，Biomni 能访问最新文献（可能超出评判模型的封闭知识），导致评判不公平。未与 OpenTCM（最近的图-RAG 系统）直接定量比较。
- **未测试安全性**：论文声明系统仅用于假设生成，不用于医疗建议，但未进行毒性或安全性评估。
