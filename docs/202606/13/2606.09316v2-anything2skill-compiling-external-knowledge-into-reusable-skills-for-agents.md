## 摘要
检索增强生成（RAG）使智能体能够在推理时访问外部知识，但其主要检索零散的陈述性证据，导致智能体需要从段落、手册、示例、日志或轨迹中反复推断任务流程。这引发了一个根本性问题：能否将从外部知识库中提取的技能安装到智能体中，使其快速近似领域专长？在本文中，我们提出了Anything2Skill，一个基于分类法引导的框架，将异构外部知识编译为可复用、可检索且可执行的智能体技能。给定一个知识记录语料库，\textsc{Anything2Skill}首先将每条记录分解为证据窗口，并在技能树先验下执行计划与扩展的技能提取。然后，将提取的候选技能转换为结构化的技能契约，明确调用条件、禁忌、动作步骤、工作流步骤、约束、输出规范、支持证据和置信度分数。为了构建可部署的程序性记忆，Anything2Skill通过分类法感知编译、注册表级协调、生命周期跟踪、版本更新和可视技能树投影，在持久化SkillBank中管理提取的技能。在推理时，智能体既从原始知识库中检索任务特定段落，也从SkillBank中检索相关程序性技能，使RAG提供陈述性证据，同时编译技能提供可复用的程序性指导。在qsv和GitHub-CLI上的实验表明，Anything2Skill结合RAG分别实现了98.85%和94.10%的成功率，显著优于仅使用RAG的智能体。这些结果表明，将潜在的程序性知识编译为显式技能是将检索增强智能体从知识访问扩展到能力复用的有效途径。

## Abstract
Retrieval-augmented generation (RAG) enables agents to access external knowledge at inference time, but it primarily retrieves fragmented declarative evidence, leaving agents to repeatedly infer task procedures from passages, manuals, examples, logs, or trajectories. This raises a fundamental question: can skills extracted from external knowledge bases be installed into an agent, enabling it to rapidly approximate domain expertise? In this paper, we propose Anything2Skill, a taxonomy-guided framework that compiles heterogeneous external knowledge into reusable, retrievable, and executable skills for agents. Given a corpus of knowledge records, \textsc{Anything2Skill} first decomposes each record into evidence windows and performs plan-and-expand skill extraction under a skill-tree prior. The extracted candidates are then converted into structured skill contracts that specify invocation conditions, contraindications, action moves, workflow steps, constraints, output specifications, supporting evidence, and confidence scores. To construct a deployable procedural memory, Anything2Skill manages the extracted skills in a persistent SkillBank through taxonomy-aware compilation, registry-level reconciliation, lifecycle tracking, versioned updates, and visible skill-tree projection. At inference time, agents retrieve both task-specific passages from the original knowledge base and relevant procedural skills from the SkillBank, allowing RAG to provide declarative evidence while compiled skills provide reusable procedural guidance. Experiments on qsv and GitHub-CLI show that Anything2Skill combined with RAG achieves 98.85\% and 94.10\% success rates, respectively, substantially outperforming RAG-only agents. These results suggest that compiling latent procedural knowledge into explicit skills is an effective way to extend retrieval-augmented agents from knowledge access toward capability reuse.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的检索增强生成（RAG）系统虽然能让智能体在推理时访问外部知识，但检索到的通常是零散、碎片化的陈述性证据（如文档段落、手册片段等）。智能体需要从这些碎片信息中反复推断和重建任务执行流程（程序性知识），导致效率低下且容易出错。核心问题在于：**能否将外部知识库中隐含的程序性知识显式地提取出来，编译成智能体可直接复用的技能模块？**
- **研究动机**：智能体在执行复杂任务时，不仅需要“知道什么”（事实），更需要“知道怎么做”（程序）。现有的RAG主要解决前者，但后者仍需模型在每次任务中自行推断。本文旨在弥补这一鸿沟，将知识库（如文档、手册、对话、日志、轨迹）中蕴含的操作流程、约束条件、动作序列等，转化为智能体可检索、可调用、可执行的技能。
- **整体含义**：本文主张智能体系统不应仅停留在“访问知识”层面，而应迈向“复用能力”阶段。通过将松散的潜在程序性知识编译为结构化的显式技能，可以显著提升智能体的任务完成能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 提出**Anything2Skill框架**，这是一个受分类法（taxonomy）引导的系统，用于将异构的外部知识记录编译成结构化、可复用、可检索、可执行的**技能契约**，并将其存储在名为**SkillBank**的持久化程序性记忆中。
- 框架旨在**扩展RAG**：传统RAG提供陈述性证据，而SkillBank提供程序性指导，两者互补。

### 关键技术细节（流程描述）
1. **证据窗口分解**：将每条知识记录（文档、手册、日志等）分解为严格定义的证据窗口，每个窗口包含文本片段、上下文标题、元数据等，确保技能的可追溯性。
2. **“计划-扩展”技能提取**：
   - **计划阶段**：使用LLM作为规划器，从每个证据窗口中识别出不同的技能候选。候选包含轻量级目标、暂定资产类型、分类节点、调用线索、禁忌、预期产物、证据提示和置信度。
   - **扩展阶段**：独立地将每个候选技能扩展为结构化技能草稿。扩展过程保守，如果窗口不支持程序性内容，则返回空草稿，避免幻觉。
   - **程序性有效性过滤**：仅保留包含至少一个程序性组件（如工作流步骤、动作、约束）且能被映射到分类法的候选。
3. **结构化技能契约**：每个技能被形式化为一个包含以下字段的元组：技能名称、描述、标准化资产类型、分类节点、调用条件、禁忌条件、动作步骤、工作流步骤、约束与注意事项、输出规范、支持证据集、置信度分数。
4. **技能管理（SkillBank构建）**：
   - **文档级编译**：使用分类法感知的身份键，将相同文档中描述同一能力的草稿分组，并生成规范化的技能规格。
   - **注册表级协调**：新技能进入SkillBank时，与现有技能进行检索匹配，并由管理层模型预测生命周期动作（如创建、强化、修订、合并、拆分、不变、丢弃）。确保跨资产层级的合并不被允许。
   - **生命周期与版本控制**：每个技能维护一个生命周期状态（候选、草稿、评估中、活跃、监视、弃用、退役），并记录版本更新。
   - **可视化技能树投影**：基于分类法，将活跃技能组织为层级化的技能树，便于导航、检索和检查。处理重复技能，进行去重与合并。
5. **推理时技能增强检索**：在推理时，智能体同时从原始知识库检索陈述性证据（RAG），并从SkillBank检索相关程序性技能，将两者作为上下文输入给下游模型。
